# ADR-001 문서 계층 구조 선택

> scope: boilerplate

## Status
accepted

## 배경
이 저장소는 Claude Code 기반의 재사용 가능한 보일러플레이트로 사용한다.
여러 프로젝트에서 반복 사용할 수 있도록 문서의 역할과 위치를 명확히 나눌 필요가 있었다.

## 결정
문서 구조를 아래와 같이 나눈다.
- 00-meta: 템플릿과 운영 가이드
- 10-charter: 프로젝트 범위와 목표
- 20-system: 시스템 구조와 설계
- 30-workitems: milestone, feature, task 단위 작업 문서
- 40-validation: QA와 개선 결과
- 90-decisions: 중요한 의사결정 기록

## 이유
- 상위 범위 정의와 하위 구현 단위를 섞지 않기 위해
- 새 프로젝트마다 일관된 문서 위치를 유지하기 위해
- 에이전트와 사람이 모두 같은 구조를 참고할 수 있게 하기 위해
- QA와 개선 문서를 구현 문서와 분리하기 위해

## 결과
이후 새 프로젝트는 이 구조를 기준으로 문서를 작성한다.
중요한 설계 선택이 생기면 별도의 ADR을 추가한다.

## 후속 작업
- `CLAUDE.md`의 경로 참조를 새 구조에 맞게 수정한다.
- skill과 subagent 설명도 새 구조를 기준으로 갱신한다.
